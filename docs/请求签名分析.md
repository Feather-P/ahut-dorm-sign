## 签名算法详解

### 算法流程

签名生成过程分为以下几个步骤：

1. **获取时间戳**
   - 使用 `new Date().getTime()` 获取当前时间戳（毫秒级）

2. **获取 Token**
   - 从本地存储中读取 Token，存储键为 `i.tokenKey`

3. **构建签名基础字符串**
   - 提取 URL 的路径部分（去除查询参数）
   - 拼接 `?sign=` 前缀

4. **生成签名值**
   - 第一次 MD5 哈希：对 `时间戳 + Token` 进行 MD5 运算
   - 第二次 MD5 哈希：对 `URL路径 + "?sign=" + 第一次哈希结果` 进行 MD5 运算
   - 拼接固定后缀 `1.` 和 Base64 编码的时间戳

5. **设置请求头**
   - `FlySource-Auth`: `bearer {token}`
   - `FlySource-sign`: `{二次哈希结果}1.{Base64时间戳}`
   - `Authorization`: `Basic {Base64(clientId:clientSecret)}`

### 签名公式

```
timestamp = 当前时间戳（毫秒）
token = 从本地存储获取的令牌

hash1 = MD5(timestamp + token)
hash2 = MD5(url_path + "?sign=" + hash1)

FlySource-sign = hash2 + "1." + base64_encode(timestamp)
```

### 哈希算法确认

通过在 `g.default` 函数处打断点，并调用 `g.default("123456")` 与标准 MD5 哈希值进行对比，确认 `g.default` 函数使用的是 **MD5 哈希算法**。

## 请求头说明

| 请求头 | 格式 | 说明 |
|--------|------|------|
| `FlySource-Auth` | `bearer {token}` | 认证令牌 |
| `FlySource-sign` | `{hash}1.{base64_timestamp}` | 请求签名 |
| `Authorization` | `Basic {base64_credentials}` | OAuth2 基础认证 |
