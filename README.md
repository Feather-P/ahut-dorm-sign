# ahut-dorm-sign

`ahut-dorm-sign` 是一个围绕 **安徽工业大学（AHUT）寝室签到场景** 的工程化项目。

项目来源于早期脚本实践，当前正在逐步重构为结构化的 **Web 后端系统**，目标是提升稳定性、可维护性与可审计性。

## 当前阶段声明

- 本项目目前处于**快速开发阶段**，接口、数据结构、配置项仍可能发生不兼容变化。
- 当前实现仅用于学习、研究与本地验证，**不具备生产可用能力**。
- **请勿将本项目部署到生产环境**，也不建议用于任何需要稳定 SLA 的正式场景。

## 项目背景

在真实校园场景中，部分同学虽然人在寝室，但可能因学习、工作、临时事务等原因，未能及时打开系统完成签到。

本项目尝试以工程化方式整理该类场景的技术实现与架构设计，核心关注点是：

- 代码结构从“脚本式”向“后端服务化”演进
- 领域逻辑、策略与基础设施解耦
- 提高可测试性、可观测性与风险控制能力

## 后续演进方向

项目未来有演进为“通用后端平台”的设想：

- 核心能力抽象为通用流程编排与任务调度；
- 不同学校/系统接入层逐步改造成插件化（Provider/Adapter）结构；
- 支持按学校差异注入认证、签名、任务拉取、签到提交等策略。

当前仓库仍以 AHUT 场景为主，上述方向属于中长期规划。

## 测试

仓库包含一个默认忽略的真实链路集成测试 [`test_school_checkin_e2e_from_env()`](tests/school_checkin_e2e.rs:26)。

如需进行“单次签到”链路验证，可按下述方式执行该测试。

### 配置方式

1. 复制 [`tests/.env.test.example`](tests/.env.test.example) 为 `tests/.env.test.local` 并填入真实值。
2. 测试会按以下优先级加载配置：
   - 环境变量 `TEST_DOTENV_PATH` 指向的文件
   - `tests/.env.test.local`
   - `tests/.env.test`
   - 若都不存在，则仅使用当前进程环境变量

### 运行方式

```bash
cargo test --test school_checkin_e2e test_school_checkin_e2e_from_env -- --ignored --nocapture
```
